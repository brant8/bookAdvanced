# StoryVerse · 产品需求文档 (PRD) v3.0

> 面向小说/漫画/动画创作者的空间化叙事构建平台

---

## 一、产品概述

### 1.1 产品定位
StoryVerse 是一个 Web 端创作工具，帮助创作者以"上帝视角"构建故事世界——从世界观搭建、故事线编排、素材管理，到剪纸动画生成与小说正文输出，打通创作全链路。

### 1.2 核心价值
- 可视化构建故事世界，替代零散笔记
- 故事线 ↔ 正文双向联动，结构与内容同步
- 剪纸风格动画一键生成，方便自媒体输出
- AI 按需介入，不强依赖，省 token 设计
- Docker 一键部署，支持本地测试与跨平台移植

### 1.3 目标用户
| 用户类型 | 使用场景 |
|---------|---------|
| 个人小说创作者 | 世界观构建、情节规划、正文写作 |
| 网文作者 | 起点/晋江风格长篇，故事线管理 |
| 漫画/动画创作者 | 分镜规划、角色素材管理、动画预览 |
| 多人编剧团队 | 协作构建世界观，分工写作 |

## 三、功能模块详述

### 3.1 世界构建层

#### 3.1.1 世界圈
- 创建"圈"作为世界容器，支持嵌套（宇宙 > 星球 > 大陆 > 城市）
- 每个圈可设置：名称、描述、封面图、访问权限
- 圈之间可建立关联关系（平行宇宙、包含关系等）

#### 3.1.2 地图系统
- **AI 生成**：文字描述 → 调用图像 API 生成地图图片
- **在线手绘**：Canvas 画板，支持自由绘制、图层、橡皮擦、颜色选择
- **上传图片**：支持用户上传自制地图
- 地图支持缩放漫游（从大陆视角到街道视角）
- 地图上可放置地点节点，节点支持点击展开详情

#### 3.1.3 地点节点
- 属性：名称、描述、氛围标签、关联角色、关联事件
- 节点间可连线，标注关系（路线/势力范围/距离）
- 节点点击可跳转到对应的故事线情节

#### 3.1.4 世界观 Lore 系统（新增）
独立的世界设定条目库，分类管理：
- **势力/组织**：门派、王国、公会等，含成员列表和势力关系
- **能力体系**：魔法规则、功法等级、道具系统，含详细规则描述
- **历史年表**：大事记时间线，可与故事内时间轴关联
- **术语表**：专有名词、地名、人名释义
- Lore 条目可直接注入 AI 生成上下文（省 token：只注入当前情节相关的条目）
- 条目之间支持互相引用和链接

---

### 3.2 故事线画布

#### 3.2.1 故事线
- 每条故事线对应一个角色或视角
- 彩色路径展示，支持自定义颜色
- 路径上挂载情节卡片（StoryNode）
- 支持主线 + 支线，支线可设置汇入主线的节点

#### 3.2.2 情节卡片（StoryNode）
每个节点包含：
```
{
  基础信息: 标题、摘要(~50字)、详细描述
  位置信息: 画布坐标、时间轴位置、故事内时间
  关联信息: 涉及角色、涉及地点、关联线索、关联Lore条目
  内容信息: 对应正文章节、段落范围
  素材信息: 关联动画帧、使用素材列表
  状态信息: 完成度、正文来源(AI/人工/混合)
}
```

#### 3.2.3 时间轴视图
- 横向时间线，所有角色故事线并排显示
- 支持拖拽调整情节时间位置
- 双击时间节点跳转到对应正文章节
- 可设置故事内时间（公元纪年/架空纪年自定义）
- 历史年表事件可叠加显示在时间轴背景层

#### 3.2.4 人物关系图
- 独立视图层，节点为角色
- 连线标注关系类型（敌对/友好/恋人/血缘/师徒等）
- 关系强度可视化（线条粗细）
- 点击角色节点展开角色详情

---

### 3.3 角色系统（扩充）

#### 3.3.1 Character Sheet
每个角色的完整设定卡：
- **基础**：姓名、别名、年龄、性别、势力归属
- **外貌**：外貌描述、avatar 素材绑定
- **性格**：性格标签、口头禅、行为禁忌
- **背景**：身世背景、重要经历
- **能力**：技能/能力列表，可引用 Lore 能力体系条目
- **语气锁定**：提供 2-3 句该角色典型对话样本，AI 生成该角色台词时自动引用
- **bio_compressed**：100字以内压缩版简介（系统自动维护，AI 调用时使用）

#### 3.3.2 人物关系图
同 3.2.4，与 Character Sheet 双向联动

---

### 3.4 素材管理

#### 3.4.1 素材来源
- 用户直接上传（PNG/JPG/WebP）
- AI 在线生图（调用图像生成 API）
- 从其他 AI 平台生成后上传

#### 3.4.2 剪裁工作台
- 用户上传含多角色/元素的图片
- 在线框选任意区域，命名归档
- 一键抠图（调用 remove.bg API 或 REMBG 模型）
- 支持在一张图内框选多个区域，批量生成素材

#### 3.4.3 素材库
分类管理：角色（多表情/多服装变体）/ 背景场景 / 建筑道具 / 特效元素
每个素材记录：名称、分类、标签、关联角色/地点、使用次数

---

### 3.5 动画引擎（剪纸风格）

#### 3.5.1 动画类型
基础动作库（MVP）：
- 位移类：slide_in / slide_out / slow_pan / cross_to
- 强调类：shake / zoom_in / zoom_out / float
- 过渡类：fade / wipe / cut / dissolve
- 组合类：enter_and_stop / chase_scene

#### 3.5.2 风格系统
预设主题（一键切换）：
- 传统剪纸（红色纸纹背景，阴影效果）
- 水墨风（宣纸纹理，水墨晕染过渡）
- 现代简约（纯色背景，无纹理）
- 用户自定义（上传背景纹理）

#### 3.5.3 AI 动画指令
AI 输出结构化 JSON，前端动画引擎解析执行：
```json
{
  "scene": "001",
  "duration": 3.5,
  "layers": [
    { "asset": "bg_forest", "action": "slow_pan_right", "speed": 0.3 },
    { "asset": "char_hero", "action": "slide_in_left", "delay": 0.5 }
  ],
  "transition": "fade_to_black",
  "caption": "他终于找到了仇人……"
}
```

#### 3.5.4 播放器
- 类视频播放器 UI（播放/暂停/进度条/章节跳转）
- 时间轴驱动，按故事线顺序自动播放
- 倍速播放（0.5x / 1x / 1.5x / 2x）

#### 3.5.5 导出规格
| 格式 | 用途 | 实现 |
|------|------|------|
| MP4 横屏 | B站/YouTube | FFmpeg.wasm（前端） |
| MP4 竖屏 | 抖音/小红书 | FFmpeg.wasm（前端） |
| MP4 方形 | Instagram | FFmpeg.wasm（前端） |
| WebM | 网页嵌入 | 浏览器原生 |
| TS | 自媒体拼接 | FFmpeg.wasm（前端） |
| GIF | 社交预览 | 前端生成 |

分辨率：横屏 1920×1080 / 竖屏 1080×1920 / 方形 1080×1080
后台异步导出接口预留，接口实现但暂不部署。

### 3.6 正文编辑器

#### 3.6.1 写作模式
| 模式 | 说明 |
|------|------|
| 🤖 AI 全写 | 选章节 → AI 根据情节卡片生成全文 |
| ✍️ 人工全写 | 纯手写，AI 不介入 |
| 🤝 AI 辅助优化 | 选中段落 → AI 润色/优化 |
| 🎯 上下文续写 | 选定位置 → AI 根据前后文续写 |

#### 3.6.2 操作粒度
- 章节级 / 段落级 / 句子级
- 右键菜单：选中文字 → AI优化 / AI扩写 / AI重写 / AI续写

#### 3.6.3 风格锁定（新增）
- 用户提供 2-3 段"参考文体"样本（粘贴到项目设置）
- AI 生成时自动在 system prompt 注入文体参考
- 支持多套风格预设（如"起点热血风" / "晋江甜文风"）
- 风格样本 hash 缓存，不重复发送

#### 3.6.4 界面设计
- 人工写作段落：白色背景
- AI 生成段落：淡蓝色背景
- 混合段落：淡紫色背景
- 每段右侧显示 token 消耗估算
- 章节列表显示完成度进度条

#### 3.6.5 AI 味检测
- 生成完成后可触发检测，标记模板化句子
- 纯前端关键词频算法为主，可选 AI 深度检测

#### 3.6.6 AI 反推功能（新增）
对现有正文/情节卡片触发：
- **情节预测**：AI 推断 3 条后续走向备选
- **漏洞检测**：发现逻辑矛盾（时间线错误、角色行为矛盾）
- **伏笔建议**：根据当前情节建议可埋入的伏笔
- 省 token：只传当前章节 + 角色 summary 链，不传全文

### 3.7 联动系统

#### 3.7.1 纯逻辑联动（无 AI，实时）
- 画布拖动情节卡片 → 正文章节自动重排
- 正文章节重排 → 画布节点位置更新
- 点击正文段落 → 画布高亮对应节点
- 时间轴拖动 → 章节顺序同步
- 新建情节卡片 → 正文新增空白章节
- 人物关系图新增角色 → 素材库新增角色槽
- 地点节点新增 → 地图自动出现标记
- 情节卡片标记"关键场景" → 时间轴出现动画标记
- Lore 条目关联情节 → AI 生成时自动注入该条目

#### 3.7.2 AI 联动（用户主动触发）
- 结构变更后衔接检测
- 情节卡片 → 生成正文
- 正文手写 → 解析出新节点
- 线索自动归类总结
- AI 反推情节走向

#### 3.7.3 AI 介入级别
```
○ 手动模式（默认）  AI 只在用户主动点击时介入
○ 提示模式         结构变更后显示"AI建议"横幅，用户点击才执行
○ 自动模式         AI 实时监听（消耗 token，用户确认后开启）
```

### 3.8 灵感速记（新增）

类似创作者"收件箱"，不需要先建节点：
- 任意时刻快速记录碎片想法（快捷键 `Cmd+Shift+N` 呼出）
- 每条灵感可标记标签（人物/情节/世界观/对话/其他）
- 状态管理：未整理 / 已关联 / 已废弃
- 可拖拽关联到具体情节卡片、角色、或 Lore 条目
- AI 辅助：一键将多条相关灵感合并整理成情节摘要

### 3.9 小说导入解析

#### 3.9.1 输入方式
- 粘贴文本 / 上传文件（.txt / .md / .docx）
- URL 解析（起点/晋江等，需用户勾选版权声明）

#### 3.9.2 版权合规

- Admin 后台一键关闭 URL 解析功能
- 解析结果页常驻版权提示

#### 3.9.3 解析输出
自动生成：角色表 / 地点节点 / 故事线 / 时间轴 / 线索库

### 3.13 版本历史（细化）

#### 正文版本
- 正文每次保存自动快照（每 5 分钟或手动保存触发）
- 可对比任意两个版本的差异（diff 视图，增删用颜色标注）
- 一键恢复到任意历史版本

#### 结构版本
- 故事线/节点结构变更单独记录快照
- 快照附带变更描述（如"调整第三章顺序"）
- 可回滚到任意结构快照

#### 存储策略（省空间）
- 正文快照只存增量 diff，不存全量文本
- 超过 90 天的快照自动合并为周快照

## 五、AI 服务设计

### 5.1 API 策略
- **用户自带 Key**：优先使用，不消耗积分
- **平台预制 API**：积分制，按 token/图片数计费
- 两种模式可混用（文字用自带，图片用平台积分）

### 5.2 省 Token 核心设计

#### 上下文窗口控制
```
AI 操作时实际发送内容：
  ├── 系统 prompt：角色 bio_compressed（~200 token）
  ├── 相关 Lore 条目（只注入关联条目，~100 token）
  ├── 风格锁定样本（hash 缓存复用，~150 token）
  ├── 前文：选中位置往前 500 字
  ├── 后文锚点：选中位置往后 200 字
  └── 任务指令

  总计 ~900 token（非全文 5000+ token）
```

#### 摘要缓存
- 每个 StoryNode 维护 summary 字段（~50字）
- AI 操作默认传 summary 链，不传正文
- 系统 prompt 组成部分 hash 缓存 24 小时

#### 批量合并
- 操作 debounce 300ms，合并连续请求
- 多人协作同类请求合并为一次调用

## 六、多人协作

### 6.1 权限体系
| 权限级别 | 操作范围 |
|---------|---------|
| 所有者 | 全部权限 + 删除项目 + 管理成员 |
| 编辑者 | 读写所有内容 |
| 评论者 | 只读 + 添加评论 |
| 只读 | 仅查看 |

### 6.2 实时协作
- Yjs（CRDT 算法）无冲突合并
- WebSocket 推送，操作 debounce 300ms 合并后推送
- 在线成员头像显示，光标位置同步
- 离线编辑缓存到 IndexedDB，网络恢复后自动同步

## 八、Docker 部署（新增）

### 8.1 部署目标
- 本地开发：`docker compose up` 一键启动全套环境
- 自托管生产：同一套 compose 文件，配置环境变量即可上线
- 跨平台：支持 Linux / macOS / Windows（WSL2）

### 8.2 服务拆分
```
docker-compose.yml 包含以下服务：
  ├── web          前端 React + Vite（Nginx 静态托管）
  ├── api          后端 Node.js + Hono
  ├── postgres     PostgreSQL 15
  ├── redis        Redis 7（队列 + 缓存）
  ├── ws-server    Yjs WebSocket 协作服务
  ├── minio        S3 兼容对象存储（本地替代 R2/S3）
  └── nginx        反向代理，统一入口
```

### 8.3 环境切换
- `docker-compose.yml`：本地开发（热重载、verbose 日志）
- `docker-compose.prod.yml`：生产覆盖（压缩镜像、健康检查、重启策略）
- `.env.example`：完整环境变量模板，含注释说明

### 8.4 数据持久化
- PostgreSQL 数据挂载到 `./data/postgres`
- MinIO 文件挂载到 `./data/minio`
- Redis 数据挂载到 `./data/redis`

### 8.5 迁移到云端
本地 → 云端迁移路径：
- PostgreSQL → Supabase / Neon / Railway（只需修改 `DATABASE_URL`）
- MinIO → Cloudflare R2 / AWS S3（只需修改 S3 配置项）
- WebSocket → 云端 WebSocket 服务（修改 `WS_SERVER_URL`）

## 十一、Story Bible（创作圣经系统）Phase 1

### 设计目标
统一管理项目级创作规则，解决 AI 风格漂移、人物设定遗忘、世界观矛盾、长篇失控等问题。是所有 AI 生成调用的"基础宪法"。

### 功能内容
- **世界规则**：底层设定约束（如"核能修士无法修炼水系功法"）
- **写作风格**：文风要求（如"起点爽文、第三人称、节奏快、避免大量心理描写"）
- **禁止事项**：禁止穿越、禁止系统流、禁止后宫等明确边界
- **角色规则**：主角底线、成长弧线约束
- **长期剧情目标**：结局方向、核心主题（如"主角最终建立星际宗门"）
- **每项支持富文本编辑，长度不限**

### AI 联动
所有 AI 生成正文的 Prompt 固定注入顺序：
```
Story Bible（规则摘要，~150 token）
+ World Rules（相关规则，~100 token）
+ Lore（关联条目）
+ Character Summary
+ StoryNode
```
Story Bible 摘要 hash 缓存，同项目内 24 小时内复用，不重复发送。

## 十二、伏笔管理系统（Foreshadow System）Phase 1

> 升级并替换现有"线索库"模块，功能为线索库超集。

### 功能内容
- 创建伏笔条目，记录：标题、描述、重要度（低/中/高）
- **双锚点**：`planted_node_id`（埋设节点）+ `reveal_node_id`（揭晓节点）
- 状态追踪：planned（计划中）/ planted（已埋设）/ revealed（已揭晓）
- **AI 超时预警**：高重要度伏笔超过 20 章未出现，自动在仪表盘提示
- 伏笔地图视图：以时间轴为底，展示所有伏笔的埋设-揭晓跨度
- 可从情节卡片右键快速创建/关联伏笔

### 省 token 设计
AI 检测伏笔状态时，只传伏笔列表（title + status + planted_chapter + 当前章节号），不传正文内容。

## 十三、世界规则引擎（World Rule Engine）Phase 1（轻量版）

### 设计目标
让 AI 生成内容前先做规则合规检查，减少世界观矛盾。

### 功能内容
- 创建规则条目：条件（condition）→ 结果（result）+ 优先级
- 示例：`修炼核能体系 → 无法使用水元素技能`
- 规则分类：能力规则 / 地理规则 / 社会规则 / 时间规则
- AI 生成前自动注入相关规则（按节点关联的 Lore/角色筛选，不全量注入）
- 规则违规高亮：AI 生成结果可选"规则检查"，输出违规项列表

## 十四、战力系统（Power System）Phase 2

### 适用场景
修仙、玄幻、科幻、热血战斗题材

### 功能内容
- 项目级战力等级表（PowerRank）：自定义等级名称、等级序号、描述
  - 示例：练气期(1) → 筑基期(2) → 金丹期(3) → 元婴期(4)
- Character Sheet 扩展字段：当前境界、战斗力数值、能力标签
- **战力成长记录**：按故事节点记录角色境界变化历史
- **AI 一致性检查**：
  - 境界提升速度异常检测
  - 技能与境界不符检测
  - 战力崩坏检测（低境界胜高境界无合理解释）

## 十六、AI 审稿人系统 Phase 2

### 四种审稿模式
| 模式 | 关注点 |
|------|--------|

```json
}
```

### 省 token 设计
只传章节摘要链 + Story Bible 摘要，不传全文。用户手动选择审稿范围（单章/多章/全文）。

## 十七、AI 角色聊天系统 Phase 2

### 设计目标
让作者直接与虚构角色"对话"，测试角色立体度，辅助角色塑造，收集灵感。

### 功能形式
- 侧边栏面板，不影响主界面
- 选择角色后进入对话模式
- 角色回复基于：Character Sheet + Lore + Story Bible
- 语气自动应用角色的 voice_samples（语气锁定样本）

### 用途
- 验证角色是否言行一致
- 测试角色面对特定情境的反应
- 从对话中提取新的灵感素材（可一键保存为灵感速记）

### 省 token 设计
每次对话只传：角色 Character Sheet + 相关 Lore + 最近 6 条对话历史，不传故事正文。
