# StoryVerse AI 原生升级路线图

> 状态：已确认的架构调整基线
> 更新日期：2026-06-14
> 目标：把现有本地创作 MVP 升级为 AI 深度参与、人工可控、本地与 NAS 优先的平台。

## 1. 审计结论

当前项目已经具备可继续演进的工程基础，不需要推倒重写：

- React、Hono、PostgreSQL、Drizzle、共享 Zod 契约和模块化单体可以保留。
- Project、Story Bible、Lore、Character、Storyline、StoryNode、Chapter 等核心实体可以扩展。
- Docker Compose、本地存储优先、NAS 部署和未来 AWS 适配方向保持不变。

但现有版本只是“结构化写作 MVP”，与 AI 原生创作平台仍有明显差距，需要一次跨数据库、
API、前端工作台和后台任务的大范围升级。

## 2. 当前功能差距

| 能力                            | 当前状态                                              | 结论                                 |
| ------------------------------- | ----------------------------------------------------- | ------------------------------------ |
| 注册、登录与会话                | 无；API 固定使用唯一的本地默认用户                    | NAS 或公网使用前必须补齐             |
| AI 设置与模型选择               | 仅正文页临时填写文本 Provider；地址和模型保存在浏览器 | 需要统一设置中心和服务端密钥管理     |
| AI 章节生成                     | 已实现故事节点驱动的候选章节与推进报告                | 保留并迁入统一生成框架               |

| 故事画布                        | 节点可拖动；连线只是按数组顺序自动绘制                | 不能满足主动连线和复杂剧情图         |
| 画布缩放、平移、小地图、框选    | 未实现                                                | 使用专业节点编辑器重构               |
| 时间轴右侧检查器                | 未实现                                                | 新增人物、场景、技能和素材配置       |
| 图片 Provider 与生图            | 未实现                                                | 与文本 Provider 分离配置             |
| 素材库和图片关联                | 蓝图中存在，代码中未实现                              | 新增本地/NAS 素材系统                |
| 定格、剪纸动画播放              | 未实现                                                | 先做浏览器分镜播放器，再考虑视频渲染 |
| AI 任务记录、审批与重试         | 未实现                                                | 新增统一生成记录和异步任务状态       |

## 3. 新产品原则

1. AI 是贯穿创作流程的核心能力，但所有结果默认是候选内容，用户确认后才写入正式数据。
2. 无 AI 或无付费 Key 时，手工编辑、管理和导出仍然可用。
3. 文本模型与图片模型分开配置，可为规划、正文、审校、角色图、场景图分别选择模型。
4. 优先支持 Ollama、OpenAI 兼容文本接口、本地 ComfyUI 或兼容图片接口。
5. API Key 只在服务端使用，加密后存储；浏览器不能读取已保存的明文 Key。
6. 图片、分镜和动画优先保存到本地或 NAS，后续通过存储适配器迁移到 S3。
7. 涉及付费调用时先展示任务范围和预计消耗，再由用户确认。

## 4. 目标架构

```text
React Web
  |-- AI 创作助手与审批面板
  |-- Story Graph 2.0
  |-- Timeline Inspector
  |-- 素材库与分镜播放器
  v
Hono API
  |-- Auth / Session
  |-- Creative Domain
  |-- AI Orchestrator / Context Builder
  |-- Text Provider Adapter
  |-- Image Provider Adapter
  |-- Generation Job Runner
  |-- Asset Storage Adapter
  v
PostgreSQL + Local/NAS File Storage
```

首期继续使用模块化单体和进程内任务执行器。图片生成、批量分镜等长任务出现可靠性需求后，
再引入独立 Worker；不在当前阶段提前增加 Redis、BullMQ 或云服务成本。

## 5. 关键领域扩展

### 5.1 认证

- 保留本机“本地所有者模式”，方便单机离线使用。
- 增加自托管账号模式：首次启动创建管理员，之后可关闭公开注册。
- 使用安全密码哈希、HttpOnly Cookie 会话、CSRF 防护、登录限流和退出全部会话。
- 项目查询从固定本地用户改为当前会话用户，并提供本地数据绑定到管理员账号的迁移流程。

### 5.2 AI Provider 与生成编排

新增建议实体：

- `ai_providers`：文本或图片 Provider、协议、地址、启用状态。
- `ai_models`：Provider 下可选模型及能力标签。
- `project_ai_profiles`：项目对规划、正文、审校和图片任务的模型选择。
- `encrypted_secrets`：加密保存 API Key。
- `generation_runs`：任务类型、输入摘要、状态、输出、错误、耗时和费用估算。

统一任务类型至少覆盖：

- 创作圣经生成、补全和一致性检查。
- 世界设定、规则和 Lore 批量生成。
- 角色、关系、外貌、能力和角色弧生成。
- 故事线、里程碑、节点和分支建议。
- 章节生成、续写、改写、审校和连续性检查。
- 角色图、场景图、背景图和分镜图生成。

所有结构化结果使用共享 Zod 契约校验，并经过“预览差异 -> 用户确认 -> 写入”的审批流程。

### 5.3 Story Graph 2.0

新增 `story_node_edges`，至少包含：

- 起点、终点、关系类型、标签、条件和排序。
- 关系类型支持主流程、分支、并行、因果、伏笔和回收。

画布采用 `@xyflow/react`。它是 MIT 开源组件，原生提供节点拖动、主动连线、缩放、平移、
多选、Controls 和 MiniMap，符合类似 Packet Tracer 的交互目标。

首版验收：

- 从节点端口拖线到另一节点并持久化。
- 删除、重连、标记和筛选边。
- 鼠标滚轮缩放、拖动画布、适配视图和小地图。
- 节点位置、画布视口和选择状态可恢复。
- 防止非法自连接，并对循环和无出口分支给出提示。

### 5.4 时间轴与右侧检查器

时间轴继续共享 StoryNode，但增加右侧检查器：

- 节点详情、故事时间、目标、冲突和状态。
- 参与人物及其当前服装、情绪、位置和状态。
- 场景、地点、天气、时间段、氛围和视觉描述。
- 角色能力/技能及本场景使用情况。
- 角色图、场景图、道具图和背景图。

建议新增 `scenes`、`character_abilities`、`story_node_character_states` 和素材关联表，
避免把所有视觉与状态数据塞进 StoryNode 的 JSON 字段。

### 5.5 素材与图片生成

新增建议实体：

- `assets`：文件、媒体类型、尺寸、来源和本地路径。
- `asset_links`：素材与角色、场景、节点、Lore 的关联。
- `image_generation_jobs`：提示词、模型、尺寸、种子、状态和结果。
- `visual_style_profiles`：项目统一画风、负面提示词和角色一致性参考。

文本 API Key 与图片 API Key 分开管理。素材默认写入 `data/uploads` 或 NAS 挂载目录，
未来只替换 Storage Adapter 即可迁移到 S3。

### 5.6 定格与剪纸动画

第一阶段不直接生成完整视频，而是建立可编辑的浏览器分镜播放器：

1. AI 根据故事线和章节生成镜头列表。
2. 用户确认每个镜头的角色、场景、旁白、时长和转场。
3. 系统生成或关联缺失图片。
4. 播放按钮按镜头顺序播放平移、缩放、淡入、剪影和分层移动效果。

建议新增 `storyboards`、`storyboard_shots`、`shot_layers` 和 `shot_transitions`。
浏览器播放优先使用 CSS 与 Web Animations API；后续有预算和明确导出需求时再接入 FFmpeg
或独立渲染 Worker。

## 7. 大范围改善判断

需要大范围改善，但不需要重写工程。

应保留基础设施和现有 CRUD 服务，重点重构以下四条主链：

1. 固定本地用户改为可切换的本地所有者/账号会话。
2. 单一章节生成改为统一 AI Provider、任务编排、审批和记录。
3. 原生手写画布改为显式边模型和 React Flow 节点编辑器。
4. 简单时间轴改为场景检查器、素材系统和分镜播放链路。

这四条链完成后，StoryVerse 才符合“AI 深度结合的小说与视觉叙事创作平台”定位。
